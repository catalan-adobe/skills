# Adding a New Skill

Step-by-step process for adding a new skill to the repository.
Covers file creation, tessl setup, manifest updates, and local sync.

## 1. Create the skill directory

```bash
mkdir -p skills/<name>/references  # if skill uses reference files
mkdir -p skills/<name>/scripts     # if skill uses helper scripts
```

Allowed directories inside a skill: `scripts/`, `references/`, `assets/`.
tessl warns about files outside these directories.

## 2. Write the skill files

- `skills/<name>/SKILL.md` — main prompt (under 500 lines)
- `skills/<name>/references/*.md` — reference docs (loaded on demand)
- `skills/<name>/scripts/*` — helper scripts (if needed)

See `.claude/rules/skill-conventions.md` for structure requirements.

### Reference file linking (critical)

tessl only recognizes **markdown link syntax** as file references:
```markdown
Read [the guide](references/GUIDE.md) for details.
```

Backtick code spans like `` `references/GUIDE.md` `` are treated as
plain text and produce orphaned file warnings during lint.

### CLAUDE_SKILL_DIR fallback

SKILL.md should include a reference location section so the agent can
find reference files when `CLAUDE_SKILL_DIR` is not set:

```markdown
## Reference Location

Read references from this skill's directory. If `CLAUDE_SKILL_DIR` is
available, references are at `${CLAUDE_SKILL_DIR}/references/`. Otherwise,
use Glob to find `skills/<name>/SKILL.md` and derive the path.
```

For scripts, use the standard fallback pattern:
```bash
if [[ -n "${CLAUDE_SKILL_DIR:-}" ]]; then
  SCRIPT="${CLAUDE_SKILL_DIR}/scripts/<script>"
else
  SCRIPT="$(command -v <script> 2>/dev/null || \
    find ~/.claude -path "*/<name>/scripts/<script>" -type f 2>/dev/null | head -1)"
fi
```

## 3. Import into tessl (creates tile.json)

New skills need a `tessl skill import` before lint/publish works:

```bash
tessl skill import --workspace catalan-adobe --public skills/<name>
```

This creates `skills/<name>/tile.json`. The file is gitignored
(`skills/*/tile.json` in `.gitignore`) — it's a local tessl artifact,
not committed to the repo.

## 4. Lint

```bash
tessl skill lint skills/<name>
```

Must pass with zero warnings. Common failures:
- **Orphaned file warning** — reference files use backtick code spans
  instead of markdown links (see step 2)
- **tile.json not found** — run `tessl skill import` first (step 3)
- **Description over 1024 chars** — shorten the `description` field in
  SKILL.md frontmatter

## 5. Review (optional but recommended)

```bash
tessl skill review skills/<name>/SKILL.md
```

Scores three dimensions: Validation, Description/Activation,
Content/Implementation. Target 70%+ on all three. Use `--optimize`
to auto-improve.

## 6. Update plugin manifests

Edit `.claude-plugin/plugin.json`:
- Add `<name> (short purpose)` to the `description` string
- Add relevant keywords to the `keywords` array

Edit `.claude-plugin/marketplace.json`:
- Add `<name>` to both `description` fields (top-level and plugin entry)

## 7. Update documentation

Edit `.claude/CLAUDE.md`:
- Add row to the "Available Skills" table:
  `| \`<name>\` | Short purpose description |`

Edit `README.md`:
- Add a new `### <name>` section under "Available Skills" with
  description, dependencies (if any), and link to SKILL.md

## 8. Sync locally

```bash
./scripts/sync-skills.sh
```

Copies the full skill directory to `~/.claude/skills/<name>/` (the
native personal skills path). Claude Code discovers skills there
automatically, sets `CLAUDE_SKILL_DIR`, and makes them available as
`/<name>` in all projects.

## 9. Commit

Steps 6-8 should be a single commit after the skill implementation
commits:

```bash
git add .claude-plugin/plugin.json .claude-plugin/marketplace.json \
  .claude/CLAUDE.md README.md
git commit -m "feat(<name>): update plugin manifests and docs"
```

Do NOT commit `tile.json` — it's gitignored.

## 10. Publish (after PR merge)

First-time publish needs `--workspace` and `--public`:

```bash
tessl skill publish --workspace catalan-adobe --public --bump patch skills/<name>
```

Subsequent publishes:

```bash
tessl skill publish --bump patch skills/<name>
```

See `.claude/rules/publishing.md` for the full publish checklist.

## Quick reference: all files touched when adding a skill

| File | Action |
|------|--------|
| `skills/<name>/SKILL.md` | Create |
| `skills/<name>/references/*.md` | Create (if needed) |
| `skills/<name>/scripts/*` | Create (if needed) |
| `skills/<name>/tile.json` | Generated by tessl (gitignored) |
| `.claude-plugin/plugin.json` | Add to description + keywords |
| `.claude-plugin/marketplace.json` | Add to descriptions |
| `.claude/CLAUDE.md` | Add to skills table |
| `README.md` | Add skills section |
